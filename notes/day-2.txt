@NgModule decorator:
-------------------------------

@NgModule({
  //register the services
  providers: [],

  //register the components, directives and pipes ONLY
  declarations: [P1, C1, D1],
  
  //register feature modules of the app and built-in modules
  imports: [DM],

  //register the name of components, pipes, directives, feature modules (already registered with declarations and imports) that you must expose outside this module. it is a subset of what you have already registered with declarations and imports (NOT services).
  exports: [P1,C1,D1,DM],

  //mention the name of bootstrapper component(s) here (must be first registered with declarations or components from some other module, whereas the that module is registered with this module in imports)

})

components are exposed by Angular as custom HTML tag. Angular runtime takes care of understanding that custom HTML tag and then injects the corresponding component's template wherever the custom HTML tag has been used.

Bootstrapper component(s): the component(s) whose template will be injected inside the index.html file by default when the app is loaded in the browser

platformBrowser function:
----------------------------------
A factory function that returns a PlatformRef instance associated with browser service providers.

PlatformRef:
----------------------------
The Angular platform is the entry point for Angular on a web page. 
Each page has exactly one platform. 
Services (such as reflection) which are common to every Angular application running on the page are bound in its scope.
method: bootstrapModule(<root-module-name>) => loads the bootstrapper module (root) and using reflection extracts metadata about that module and then creates instance of it, loads associated registered assets...

BrowserModule:
-------------------
Exports required infrastructure for all Angular apps. Included by default in all Angular apps created with the CLI new command. 
It needs to be imported ONLY in the root module file (one time)
Note: Re-exports CommonModule and ApplicationModule, making their exports and providers available to all apps.

Binding:
-----------------
a. data binding: to display some data from the component code in the component's template
flow of data: from code to template

i. interpolation
syntax: <element attr={{class-prop-name}}> => converts the value of the property to string automatically before binding

ii. property binding:
syntax: <element [attr]="prop-name"> => does not convert the value of the property to string. used in numerical value binding, object binding (in case of form handling etc.)

b. event binding: binding an event of an element in template to a method in component class code. to pass data from template to code.
flow of data: from template to code.
syntax: <element (event-name)="method-in-class()">

change tracking or change notification: 
-----------------------------------------
A mechanism where Angular checks the changes in the property values and accordingly update everywhere the property data has been used, including the template

$event => built-in property (object) which represents the event related argument that was emitted when the event was fired by user of a control/tag
$event => Event object/ some other value (whatever you are emitting when the event is fired)
"target" => source of the event (any tag)


passing data between components:
-----------------------------------------
a. when components are nested inside each other
---------------------------------------------------
	i. shallow nesting (immediate nesting)
	------------------------------------------------

		A. from parent to child: data binding
		------------------------------------------
		Note: declare a property in the child class code with @Input() decorator
		bind the property with the parent data in the template of paremt component where the child component is nested

	@Input():
	--------------
	Decorator that marks a class field as an input property and supplies configuration metadata. The input property is bound to a DOM property in the template. During change detection, Angular automatically updates the data property with the DOM property's value.
	
		B. from child to parent: event binding
		-------------------------------------------
		Note: declare a property in the child component and assign the property to an instance of EventEmitter<T> where T is the type of the data that you want emit when the event is fired
		fire the event at an appropriate time and emit the value
		in the parent template, where the child component is nested, handle that event of the child by attaching an event handler (method) from the parent component code
	
	EventEmitter: 
	---------------
	Use in components with the @Output directive to emit custom events synchronously or asynchronously, and register handlers for those events by subscribing to an instance.

	@Output():
	------------------
	Decorator that marks a class field as an output property and supplies configuration metadata. The DOM property bound to the output property is automatically updated during change detection.
